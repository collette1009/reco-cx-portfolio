누락과 불일치를 찾아 사실관계를 맞추고,
처리 완료까지 관리해왔습니다.

출고·미출·송장 데이터가 실제 고객 안내와 후속 처리로 이어지는지 확인하고, 누락·미연결 구간은 유관부서와 조정했습니다. 반복되는 검수·입력 구간은 기준과 도구로 정리했습니다.

초록마을에서는 3PL 전환 초기 미출 리스트 미공유, 주문번호·송장번호 미연결, 회원코드 미노출 등 운영에 필요한 데이터 공백을 확인하고, 임시 조회 방식과 후속 처리 기준을 정리했습니다. 반복 입력 구간은 자동화해 제품 기준 1,151품목의 재배송 전산 처리를 완료했습니다. 카카오VX에서는 400여 곳 제휴 골프장의 운영 정보를 전수 검수하고, 누락되거나 불명확한 내용은 제휴사에 전화로 확인해 보완했습니다.

CASE 01
운영 데이터 공백 대응유관부서 협의대량 처리 자동화

3PL 전환 초기 출고 데이터 공백 대응 및 1,151품목 재배송 처리

내 역할 · 출고·미출 현황 확인, 고객 안내·취소·재배송 기준 정리, 데이터 누락 구간 식별, 물류·EC·정보전략·개발 담당자 협의, 임시 조회 구조 운영, 반복 입력 자동화 및 후속 물량 관리

미출 리스트·송장번호·회원코드가 실제 후속 업무에 연결되지 않는 구간을 확인하고, 임시 조회 방식과 시스템 개선 요청을 병행하며 제품 기준 1,151품목의 재배송 전산 처리를 완료했습니다.

Problem

3PL 전환 초기 출고 일정과 실제 전산 정보가 안정적으로 연결되지 않았습니다. 미출 주문이 정확한 리스트로 사전 공유되지 않거나 송장번호가 주문 데이터와 연결되지 않아 고객과 내부 담당자가 배송 상태를 확인하기 어려웠습니다. CRM 출고리스트에는 보상 대상 가공에 필요한 회원코드가 없어 주문별 개별 조회가 필요했고, 대량 재배송 또한 일괄 처리 기능이 없어 수기 처리 부담이 큰 상황이었습니다.

Action

1
출고 상황별 처리 기준 정리
도착희망일과 출고 가능 여부, 고객 요청에 따라 지연 안내·재배송·취소 기준을 구분했습니다. 출고 여부가 확정되지 않은 주문은 안내를 보류하고, 확정 시점과 후속 처리 일정을 담당자에게 재확인했습니다.
2
운영에 필요한 데이터 누락 구간 확인
미출 리스트 미공유, 송장번호 미연결, 회원코드 미노출로 인해 발생하는 운영 부담을 확인하고, 각 데이터가 필요한 업무와 후속 영향을 정리해 유관부서에 공유했습니다.
3
후속 영향을 고려해 처리 방안 변경
전체 미출 처리가 재배송 프로세스에도 영향을 줄 수 있음을 확인한 뒤 요청을 보류하고, 보상 대상 가공에 필요한 회원코드를 CRM 출고리스트에 직접 노출하는 방식으로 대안을 전환했습니다.
4
시스템 정상화 전 임시 운영방안 적용
송장번호 연결이 정상화되기 전에는 3PL 파트너로부터 전달받은 운송장 엑셀을 가공해 택배사 배송조회 링크를 생성하고, 개별 배송 상태를 조회할 수 있도록 운영했습니다.
5
대량 반복 처리 자동화
일괄 재배송 기능이 없는 상황에서 Chrome Extension과 Google Sheets Apps Script를 연동해 반복 입력 구간을 단축하고, 제품 기준 1,151품목의 재배송 전산 처리를 완료했습니다. 처리 판단이 필요한 주문은 담당자가 직접 확인하도록 남겼습니다.

Result

CRM 출고리스트에 회원코드가 추가되어 별도 어드민 조회 없이 보상 대상 자료를 가공할 수 있게 됐습니다. 송장번호 연결 전에는 운송장 파일 기반 임시 조회 방식을 운영했고, 주문번호와 송장번호가 연결되지 않은 원인을 확인해 개발·물류 담당자의 후속 조치로 연결했습니다. 반복 입력 구간을 자동화해 제품 기준 1,151품목의 재배송 전산 처리를 완료하고, 남은 취소 대상과 처리 일정을 다시 산정해 후속 작업을 관리했습니다. 이후에도 동일 도구를 반품·재배송 일상 업무에 상시 적용하고 있습니다.

올바로 업무에서도 인계서 필수값과 상태가 실제 후속 처리에 연결되는지 확인하고, 누락·미연결 건은 담당자 확인과 재검수 시점까지 남겨 관리하겠습니다.

CASE 02
반복 데이터 검수전화 확인정보 표준화

400여 곳 제휴사 운영정보 전수 검수 및 누락 정보 확인

내 역할 · 제휴사 정보 전수 조사, 누락·불명확 정보 전화 확인, 공통 확인 항목과 조회 순서 정리, 업데이트 기준 문서화

골프장마다 다르게 관리되던 예약·취소·위약 정보를 공통 항목과 조회 순서로 표준화하고, 누락된 내용은 전화로 직접 확인했습니다.

Problem

카카오VX에서 제휴 골프장별 예약·취소·위약 정보가 어드민에 등록돼 있었지만, 골프장마다 정보 위치와 표현 방식이 달랐습니다. 예약실 연락처, 운영시간, 취소 가능 기한, 위약 적용 범위, 입금계좌, 동반자 제한 여부가 누락되거나 불명확한 곳도 있었습니다. 담당자가 필요한 정보를 찾는 순서도 제각각이어서 같은 문의에도 확인 시간과 판단 방식이 달라질 수 있었습니다.

Action

1
400여 곳의 어드민 정보를 순서대로 전수 확인
2
반복 확인이 필요한 내용을 공통 항목으로 정리
3
누락되거나 의미가 불명확한 내용은 골프장에 직접 전화 확인
4
정보 위치와 표현 순서를 통일
5
복합·예외 케이스를 FAQ와 판단 기준으로 정리
6
정책 변경 시 수정해야 할 항목과 공유 방법 문서화

Result

400여 곳 제휴 골프장의 운영 정보를 공통 항목과 일정한 조회 순서로 정리했습니다. 누락·불명확한 내용은 제휴사에 전화로 확인해 보완했고, 담당자가 달라도 같은 순서로 정보를 확인할 수 있게 만들었습니다.

그 외 사례
반복 이슈를 입력·안내 구조 개선으로 연결
01

문의 입력 구조 개선

자유 텍스트에 섞여 있던 환불·교환 의도를 선택값으로 분리하고, 해당 값이 CRM에 노출되도록 개선했습니다. 담당자가 고객 의사를 다시 확인하거나 임의로 판단하는 과정을 줄였습니다.

02

장바구니 안내 구조 개선

일 6~10건 반복되던 배송 제약 문의의 원인을 안내 구조에서 확인하고, 장바구니에서 원인 상품과 배송 가능 요일을 볼 수 있도록 개선했습니다. 반영 후 관련 문의는 0건 수준으로 감소했습니다.

CASE 03

3PL 사고·피해현황 및 정산 협의 자료 관리

사고·피해현황 관리정산 협의물류 파트너 커뮤니케이션
내 역할 · 3PL 사고·피해 유형 분류, 월별 피해현황 정리, 물류 계약 조항 확인, 물류 담당자 정산 협의

Problem

3PL 전환 이후 출고 누락·포장 불량·파손 등 파트너 귀책 가능 사고가 발생했지만, 회사가 먼저 처리한 환불·재배송·보상 비용을 어떤 기준으로 정리하고 정산에 반영할지 실무 기준이 명확하지 않았습니다.

Action

1
월별 온라인 피해현황을 정리하고 피해 유형, 회사 선처리 비용, 차감 예정액, 확인 필요 항목을 구분
2
물류 계약서 조항을 기준으로 정산 반영 가능 여부를 확인
3
이견이 있는 항목은 물류 담당자와 재확인

Result

파트너 사고 현황과 정산 검토 항목을 같은 자료에서 확인할 수 있게 했고, 월별 정산 협의 자료로 활용했습니다.

별도 택배사 협의 사례
택배 사고 증빙·회수 기준 재협의

택배사 사고 기록 유무와 상품의 안전·위생 상태에 따라 증빙·회수 기준을 구분하도록 협의했고, 이후 월 배상 회수액은 전년 동월 대비 약 14.7배 증가했습니다.

CASE 04

담당자가 달라도 같은 방식으로 처리하기 위한 운영 기준과 교육 자료 정리

운영정책매뉴얼교육·검수
내 역할 · 기존 문서와 실제 처리 사례 검토, 확인 항목·처리 순서·이관 조건 정리, 처리 방향 및 결과 검수, 온보딩 자료 제작

Problem

조직 변동으로 타부서 지원 인력이 운영 업무에 투입됐습니다. 업무 경험이 다른 담당자들이 같은 이슈를 처리하면서 확인 항목, 처리 범위, 타부서 판단이 필요한 시점이 달라질 수 있었습니다.

Action

1
기존 문서와 실제 사례를 기준으로 처리 순서 재정리
2
이슈 유형별 필수 확인 항목 작성
3
담당자가 직접 처리할 수 있는 범위와 타부서 판단이 필요한 범위 구분
4
이관 시 함께 전달해야 할 사실관계와 증빙 항목 정리
5
처리 결과를 검수하고 피드백
6
FAQ, 체크리스트, 에스컬레이션 기준, 온보딩 자료로 문서화

Result

지원 인력이 같은 확인 순서와 이관 기준을 적용할 수 있도록 운영 가이드를 정리하고 처리 결과를 검수했습니다. 담당자가 바뀌는 상황에서도 업무 기준과 후속 조치가 남도록 관리했습니다.
